# 启动说明

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置环境变量

```bash
cp env.example .env
# 编辑 .env 文件，配置必要的参数
```

### 3. 启动服务

**本地开发**:
```bash
python start_server.py
```

**生产部署**:
```bash
MODE=all ./start.sh
```

## 📋 启动文件说明

### 1. `start.sh` - 生产环境启动脚本

用于生产环境部署，支持多种启动模式。

#### 使用方式

```bash
# 方式1: Web 服务模式（默认，仅启动 FastAPI）
./start.sh

# 方式2: 完整服务模式（推荐，单容器同时启动 FastAPI + ARQ Worker）
MODE=all ./start.sh

# 方式3: ARQ Worker 模式（仅启动 ARQ Worker）
MODE=arq ./start.sh

# 方式4: Celery Worker 模式（如果使用 Celery）
MODE=worker ./start.sh

# 方式5: Celery Beat 模式（如果使用 Celery）
MODE=beat ./start.sh
```

**推荐使用 `MODE=all`** 在单容器中同时运行 FastAPI 和 ARQ Worker！

#### 环境变量

```bash
# Web 服务配置
APP_PORT=8000                    # 服务端口
SERVER_WORKER_AMOUNT=4           # Worker 数量
SERVER_WORKER_CLASS=uvicorn.workers.UvicornWorker
GUNICORN_TIMEOUT=200             # 超时时间
LOG_LEVEL=info                   # 日志级别

# 持久化配置
PERSISTENCE_ENABLED=true         # 是否启用持久化
PERSISTENCE_BACKEND=file         # 持久化后端: file 或 es
PERSISTENCE_INTERVAL=300         # 持久化间隔（秒）

# Redis 配置
REDIS_HOST=redis
REDIS_PORT=6379
```

#### Docker Compose 示例

##### 方案1: 单容器模式（推荐，简单）

```yaml
version: '3.8'

services:
  # 完整服务（FastAPI + ARQ Worker）
  app:
    build: .
    command: ./start.sh
    environment:
      - MODE=all  # 同时启动 FastAPI 和 ARQ Worker
      - APP_PORT=8000
      - PERSISTENCE_ENABLED=true
      - PERSISTENCE_BACKEND=file
      - PERSISTENCE_INTERVAL=300
    ports:
      - "8000:8000"
    depends_on:
      - redis
    volumes:
      - ./data:/app/data  # 持久化数据目录

  # Redis
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

##### 方案2: 多容器模式（推荐，可独立扩展）

```yaml
version: '3.8'

services:
  # Web 服务
  app:
    build: .
    command: ./start.sh
    environment:
      - MODE=web  # 或省略，默认就是 web 模式
      - APP_PORT=8000
      - PERSISTENCE_ENABLED=true
    ports:
      - "8000:8000"
    depends_on:
      - redis
      - arq-worker

  # ARQ Worker
  arq-worker:
    build: .
    command: ./start.sh
    environment:
      - MODE=arq
      - PERSISTENCE_ENABLED=true
      - PERSISTENCE_BACKEND=file
      - PERSISTENCE_INTERVAL=300
    depends_on:
      - redis
    volumes:
      - ./data:/app/data  # 共享持久化数据目录

  # Redis
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

### 2. `start_server.py` - 本地开发启动脚本

用于本地开发调试，自动启动 FastAPI 和 ARQ Worker。

#### 使用方式

```bash
# 直接运行
python start_server.py

# 或使用可执行权限
chmod +x start_server.py
./start_server.py
```

#### 功能特点

1. ✅ **自动启动 ARQ Worker** - 如果启用持久化，自动启动 Worker
2. ✅ **热重载** - FastAPI 支持代码热重载
3. ✅ **统一管理** - Ctrl+C 同时停止所有服务
4. ✅ **友好提示** - 显示服务地址和配置信息
5. ✅ **错误处理** - 启动失败时给出明确提示

#### 启动日志示例

```

   ✅ ARQ Worker 启动成功
```

## 🐳 Docker 部署

### 生产部署

#### 方式1: Docker Compose（推荐）

```bash
# 1. 构建镜像
docker-compose build

# 2. 启动所有服务
docker-compose up -d

# 3. 查看日志
docker-compose logs -f

# 4. 停止服务
docker-compose down
```

#### 方式2: 单容器启动（推荐）

```bash
# 同时启动 FastAPI 和 ARQ Worker
MODE=all ./start.sh
```

#### 方式3: 多终端启动

```bash
# 终端1 - 启动 ARQ Worker
MODE=arq ./start.sh

# 终端2 - 启动 Web 服务
./start.sh
```

## 🔧 常见问题

### Q1: start_server.py 提示 ARQ 未安装

**解决**:
```bash
pip install arq>=0.25.0
```

### Q2: start_server.py 无法连接 Redis

**检查**:
```bash
# 1. 确认 Redis 运行
redis-cli ping

# 2. 检查配置
grep REDIS .env

# 3. 本地启动 Redis
redis-server

# 或使用 Docker
docker run -d -p 6379:6379 redis:7-alpine
```

### Q3: start.sh 在 Docker 中无法启动

**检查**:
```bash
# 1. 确认文件有执行权限
chmod +x start.sh

# 2. 确认环境变量正确
docker-compose config

# 3. 查看日志
docker-compose logs app
```

### Q4: 如何禁用持久化功能？

**修改 .env**:
```env
PERSISTENCE_ENABLED=false
```

这样 `start_server.py` 就不会启动 ARQ Worker。
